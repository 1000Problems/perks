# TASK: Top-N ranking and "why" sentence generator

> Compose eligibility + scoring into a ranked list of recommended cards, with a one-line plain-English "why" for each.

## Context

The recommendation panel shows the top 5 cards with a delta and a one-sentence rationale. The "why" is what makes the rec feel intelligent rather than algorithmic — it tells the user what specifically is different. This is a string generator that looks at the score breakdown and surfaces the one thing that matters most for that user.

## Requirements

1. Create `lib/engine/ranking.ts` exporting:
   ```ts
   export function rankCards(
     userProfile: UserProfile,
     wallet: WalletCardHeld[],
     db: CardDatabase,
     options: RankOptions,
   ): RankResult;

   export interface RankResult {
     visible: RankedRecommendation[];
     denied: RankedRecommendation[];
   }

   export interface RankedRecommendation {
     card: Card;
     score: CardScore;
     eligibility: { status: "green" | "yellow" | "red"; note: string };
     why: string;
     rank: number;
   }
   ```
2. Iterate `db.cards`, skip cards already in `wallet` (by `card_id`), score each against the user's profile, evaluate eligibility, and split into visible (green/yellow) vs denied (red).
3. `RankOptions.filter` is one of `"total" | "payself" | "nofee" | "premium"`:
   - `total`: sort by `score.deltaOngoing` descending
   - `payself`: only cards where `deltaOngoing > 0`, sort by `deltaOngoing` desc
   - `nofee`: only cards with `annual_fee_usd === 0`, sort by `deltaOngoing` desc
   - `premium`: only cards with `annual_fee_usd >= 395`, sort by `deltaOngoing` desc
4. The "why" sentence is generated by `generateWhy(score, eligibility, userProfile, db)`. Pick the single highest-impact reason from the score breakdown. Examples (write a real generator, not a template lookup):
   - "Doubles your dining rate, adds Hyatt transfers for your Phoenix trip."
   - "Pays for itself with the Walmart+ credit you already use."
   - "Adds 5% on online shopping, your biggest category gap."
   - "Heavy on perks. Worth it only if you fly enough to use the lounges."
5. Return at most 5 cards in `visible`. Always return all denied cards in `denied`.

## Implementation Notes

- Pure function. Reuses `evaluateEligibility` (TASK-02) and `scoreCard` (TASK-03).
- Types from `@/lib/data/loader`.
- The "why" generator is the most opinionated piece. Suggested logic:
  1. If a transferable points partner unlocks a destination perk for one of the user's `trips_planned`, lead with that. Use `db.destinationPerks` to look up the user's destinations and check if `db.programById.get(card.currency_earned)` has a relevant transfer partner.
  2. Else if a single new perk has value > $100 and matches a brand the user listed in `userProfile.brands_used`, lead with that.
  3. Else if a single category jumps by 2%+ in spend impact, lead with that.
  4. Else fall back to the breakdown's largest line item.
- Keep the "why" under 90 characters — must fit on one line in the rec card.
- Add unit tests in `tests/engine/ranking.test.ts` covering: filter modes, the deny-list path, an empty candidate set, the "why" generator picking destination > brand > category > breakdown.
